If a real bug is confirmed, prepare the issue details using the available execution artifacts.

Issue format:

- Title
- Summary
- Description
- Severity
- Steps to reproduce
- Expected result
- Actual result
- Attachment
  - Screenshot path if available

Formatting rules:
- summary: 1 sentence only
- description: max 3 sentences
- keep both concise and non-repetitive
- Do not repeat the title in the description
- Do not explain test framework internals unless the failure is a setup issue
- Focus on the product bug and its user impact

After issue creation, update the local report with:
- issue URL or issue number
- screenshot path
